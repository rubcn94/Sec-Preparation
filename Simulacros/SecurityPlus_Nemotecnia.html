Pares que se confunden

Los conceptos que mas fallos generan en el examen. Aprende la diferencia clave de cada par.

IDS vs IPS

IDS
Intrusion Detection System. Monitoriza y alerta. Modo pasivo — no bloquea, solo observa.
IDS = "iD reSearch" — solo busca, no actua
IPS
Intrusion Prevention System. Monitoriza y bloquea. Modo inline — activo en el flujo de red.
IPS = "iP Stop" — para el paquete
Regla de oro: IDS es el guardia que avisa, IPS es el portero que bloquea la entrada.

SIEM vs SOAR

SIEM
Agrega y correlaciona logs. Genera alertas. Requiere accion humana para responder.
SIEM = "See 'Em" — los ves en un panel
SOAR
Automatiza la respuesta. Usa playbooks para reaccionar sin intervencion humana.
SOAR = vuela solo, actua solo
SIEM recopila e informa → tu actuas. SOAR recopila y actua → solo si esta configurado.

RTO vs RPO

RTO
Recovery Time Objective. Tiempo maximo aceptable para restaurar un sistema tras un fallo.
RTO = "Time to Operate again"
RPO
Recovery Point Objective. Maxima perdida de datos aceptable, medida desde el ultimo backup.
RPO = "Point of last backup"
RTO = cuanto tardamos en volver? · RPO = cuantos datos podemos perder?

Virus vs Worm vs Trojan

Virus
Necesita host / accion humana para ejecutarse y propagarse. Se adjunta a archivos.
Worm
Se propaga solo por la red, sin interaccion humana. Explota vulnerabilidades automaticamente.
Trojan
Se disfraza de software legitimo. No se replica solo — el usuario lo instala voluntariamente.
Troya: entraron camuflados → Trojan = camuflaje

Symmetric vs Asymmetric

Symmetric
Una sola clave para cifrar y descifrar. Rapido. Ideal para grandes volumenes de datos. (AES, DES, 3DES)
"sym" = mismo = misma clave
Asymmetric
Par de claves publica/privada. Lento. Ideal para intercambio de claves y firmas. (RSA, ECC, DSA)
"asym" = diferente = 2 claves distintas
En la practica: asymmetric intercambia la clave, symmetric cifra los datos. TLS usa ambos.

AuthN vs AuthZ

AuthN
Quien eres? Verifica identidad. (contrasena, biometria, MFA, certificados)
AutheNtication → N = iNdentidad
AuthZ
Que puedes hacer? Determina permisos. (roles, ACLs, politicas)
AuthoriZation → Z = permisos
Siempre en orden: primero AuthN (identificar), luego AuthZ (autorizar).

Threat · Vulnerability · Risk

Threat
Actor o evento que puede causar dano. (hacker, malware, desastre natural)
Vulnerability
Debilidad en el sistema que puede ser explotada. (bug, misconfiguration, falta de parche)
Risk
Risk = Threat x Vulnerability x Impact. La probabilidad de que una amenaza explote una debilidad y el dano resultante.

SLE · ALE · ARO (Formulas de riesgo)

SLE
Single Loss Expectancy
SLE = AV x EF
Perdida por un unico incidente.
ALE
Annual Loss Expectancy
ALE = SLE x ARO
Perdida esperada en un ano.
AV
Asset Value — valor del activo en euros
EF / ARO
EF = Exposure Factor (% perdida)
ARO = Annualized Rate of Occurrence
Orden: AV x EF = SLE → SLE x ARO = ALE. De activo a anual.
