D1 · General Security Concepts
~11 preguntas | Fundamentos: CIA, AAA, controles, criptografía, actores de amenaza, ingeniería social
🔒
CIA Triad + AAA ▶
Confidentiality — Solo acceden quienes deben. Cifrado, control de acceso
Integrity — Los datos no se alteran. Hashing, firmas digitales
Availability — El sistema está disponible. Redundancia, RAID, backups
Authentication — Verificar identidad (¿quién eres?)
Authorization — Qué puedes hacer (permisos, ACL)
Accounting — Registrar lo que hiciste (logs, auditoría)
🎭
Actores de Amenaza ▶
Nation-state / APT — Mayor sofisticación, recursos ilimitados, espionaje
Insider — Mayor riesgo REAL: tiene acceso legítimo. Malicioso o accidental
Crimen organizado — Motivación económica: ransomware, fraude
Hacktivista — Motivación ideológica (Anonymous). DDoS, filtraciones
Script kiddie — Bajo nivel técnico, usa herramientas de otros
Privilege creep — Acumulación de permisos por cambios de rol sin revocar
🪤
Ingeniería Social / Phishing ▶
Phishing — Email masivo falso → credenciales o malware
Spear phishing — Dirigido a persona/empresa específica
Whaling — Spear phishing a ejecutivos C-level
Smishing — Phishing por SMS
Vishing — Phishing por voz / llamada
Quishing — Phishing por QR code
BEC — Business Email Compromise: suplantar CEO para transferencias
Pretexting — Historia falsa creíble ("soy de IT, dame tu password")
Watering hole — Infectar sitio que visitan los empleados objetivo
Typosquatting — Dominio con error tipográfico (gooogle.com)
Shoulder surfing — Mirar pantalla para robar credenciales
Dumpster diving — Buscar info en la basura
🛡️
Controles de Seguridad ▶
POR FUNCIÓN:
Preventivo — Evita que ocurra. Firewall, antivirus, MFA
Detectivo — Detecta cuando ocurre. IDS, SIEM, auditoría
Correctivo — Repara después. Backup, patch, IR
Compensatorio — Sustituto temporal cuando el ideal no es viable. WAF sin parche
Disuasorio — Desalienta ataques. Cámaras, señales de advertencia
POR CATEGORÍA:
Técnico — Software/hardware: firewall, cifrado
Administrativo — Políticas, procedimientos, formación
Operacional — Procesos del día a día: revisiones, guardia
Físico — Barreras físicas: candados, jaulas, CCTV
Nemotecnia: TOFG → Técnico Odia el Frío → T·O·F·G
🔑
Criptografía ▶
Simétrica — Misma clave cifra/descifra. Rápida. AES, DES, 3DES
Asimétrica — Clave pública/privada. Lenta. RSA, ECC, Diffie-Hellman
Hashing — IRREVERSIBLE, integridad. MD5 (roto), SHA-256, SHA-3
AES — Estándar simétrico. 128/192/256 bits. FIPS 197
RSA — Asimétrico más común. 2048+ bits recomendado
ECC — Más eficiente que RSA. Misma seguridad con clave más corta
Salt — Único por usuario, público, almacenado con el hash
Pepper — Secreto global, fuera de la BD
PKI — Infraestructura de clave pública. CA, RA, certificados X.509
OCSP — Verificación en tiempo real de revocación de certificado
CRL — Lista de certificados revocados (batch)
🚫
Zero Trust + Supply Chain ▶
Zero Trust — "Never trust, always verify". No hay perímetro seguro
Least privilege — Solo los permisos mínimos necesarios
Microsegmentación — Dividir la red en zonas pequeñas aisladas
Supply chain attack — Comprometer proveedor para atacar a sus clientes
SBOM — Software Bill of Materials: lista de componentes del software
SolarWinds (2020) — Actualización maliciosa afectó 18.000 organizaciones

D2 · Threats, Vulnerabilities & Mitigations
~20 preguntas | Malware, ataques de red y app, gestión de vulnerabilidades, técnicas de ataque avanzadas
🦠
Tipos de Malware ▶
Virus — Necesita interacción del usuario. Se replica por archivos
Worm — Automático, sin interacción. Se propaga por la red
Trojan — Se hace pasar por software legítimo. No se replica solo
Ransomware — Cifra archivos y pide rescate. Doble extorsión = también exfiltra
RAT — Remote Access Trojan: control remoto del sistema
Rootkit — Oculta su presencia a nivel kernel/OS
Bootkit — Persiste en MBR/UEFI. TRAMPA Sobrevive a reinstalar el OS
Keylogger — Registra pulsaciones de teclado
Spyware — Monitorea actividad del usuario
Adware — Muestra publicidad + puede monitorear
Logic bomb — Ejecuta payload cuando se cumple condición (fecha, evento)
Fileless — Solo en RAM, sin archivos en disco. Evade AV basado en firmas
Crypto miner — Usa CPU/GPU para minar criptomonedas
Botnet — Red de dispositivos infectados (zombies) bajo C2
🌐
Ataques de Red ▶
DDoS — Desde múltiples orígenes → colapsar servicio
SYN Flood — Inunda con paquetes SYN sin completar handshake
Smurf attack — ICMP broadcast con IP spoofed de la víctima
ARP Spoofing — Falsifica tabla ARP → MITM en red local
DNS Poisoning — Envenena caché DNS → redirige a IP falsa
DNS Amplification — Spoofea IP víctima + open resolvers → DDoS ≠ DNS Poisoning
MITM — Intercepta y puede modificar comunicación
Evil Twin — AP falso con mismo SSID que red legítima → MITM
Rogue AP — AP no autorizado en la red (no necesariamente imita nada)
MAC Flooding — Satura tabla CAM del switch → tráfico a todos los puertos
VLAN Hopping — Accede a VLANs no autorizadas. Prevención: apagar DTP
💉
Ataques de Aplicación ▶
SQLi — Inyecta SQL en inputs → manipula BD (comillas simples como prueba)
Blind SQLi — Sin mensajes de error, pero comportamiento diferente EXAM
XSS Stored — Script en BD → afecta a todos los visitantes
XSS Reflected — Script en URL → requiere que víctima haga clic
CSRF — Fuerza acciones en nombre del usuario autenticado (sin su consentimiento)
SSRF — Servidor hace solicitudes internas a instancia del atacante
IDOR — Cambias ID en URL y accedes a datos ajenos ≠ SQLi
Directory Traversal — ../../../etc/passwd: accede fuera del directorio web
XXE — XML External Entity: el parser XML lee archivos del servidor
Buffer Overflow — Sobreescribe memoria adyacente → ejecución de código
Race Condition / TOCTOU — Explota ventana entre verificar y usar
🗝️
Ataques de Credenciales + Kerberos ▶
Brute Force — Prueba todas las combinaciones posibles
Dictionary attack — Lista de palabras comunes contra hashes
Password Spraying — Pocas contraseñas comunes contra muchas cuentas
Credential Stuffing — Reutiliza credenciales de brechas anteriores
Rainbow Table — Hashes precalculados. Prevenido con salting
Pass-the-Hash — Usa hash NTLM directamente sin crackearlo → Lateral Movement
Kerberoasting — Solicita TGS para SPN → crackeo offline contraseña de servicio EXAM
Golden Ticket — Forja TGT con hash de KRBTGT → acceso total al dominio
Pass-the-Ticket — Roba y reutiliza ticket Kerberos válido
🔍
Gestión de Vulnerabilidades ▶
CVSS v3.1 — Puntuación 0-10. Componentes: AV, AC, PR, UI, S, C, I, A
CVE — Identificador único de vulnerabilidad (CVE-YYYY-NNNNN)
Zero-day — Vulnerabilidad sin parche conocido. Máximo riesgo
Vuln scanning — Identifica vulnerabilidades sin explotarlas (no destructivo)
Pentesting — Explota vulnerabilidades con autorización
Patch management — Proceso de aplicar parches. Reto: compatibilidad vs rapidez
Responsible disclosure — 90 días de embargo antes de publicar
🥷
Técnicas de Evasión + LotL ▶
Living off the Land (LotL) — Usa herramientas nativas del OS para evadir detección
LOLBins — Binarios legítimos usados con fines maliciosos (certutil, mshta, wmic)
Polymorphism — Cambia código en cada replicación (misma funcionalidad)
Metamorphism — Reescribe completamente su código
Anti-VM/Anti-Analysis — Detecta entorno sandbox y duerme o cambia comportamiento
Timestomping — Modifica timestamps de archivos para ocultar actividad (Defense Evasion)
DNS Tunneling — Usa consultas DNS para comunicación C2 (difícil de detectar)

D3 · Security Architecture
~16 preguntas | Cloud, firewalls, IDS/IPS, VPN, Zero Trust Architecture, contenedores, diseño seguro
☁️
Cloud — Modelos de Servicio ▶
IaaS — Cliente: SO + apps + datos. Proveedor: hardware/hipervisor. Ej: AWS EC2 EXAM
PaaS — Cliente: apps + datos. Proveedor: SO + plataforma. Ej: Heroku
SaaS — Cliente: solo datos/acceso. Proveedor: TODO. Ej: Gmail, O365
Proveedor SIEMPRE responsable de: seguridad física del datacenter
Pública — Compartida | Privada — Exclusiva | Híbrida — Ambas | Multi-cloud — Varios proveedores
CSPM — Cloud Security Posture Mgmt: detecta misconfiguraciones en multi-cloud
CASB — Cloud Access Security Broker: visibilidad y control de apps cloud
Data sovereignty — Los datos se rigen por leyes del país donde se almacenan físicamente
🔥
Firewalls — Tipos por Capa OSI ▶
Packet filter (stateless) — Capa 3. Solo ve IP/puerto. No rastrea conexiones. El más básico
Stateful firewall — Capa 3-4. Rastrea estado TCP (SYN, ACK). No inspecciona payload HTTP
NGFW — Capa 7. Firewall + IPS + inspección de apps + TLS inspection. Ve dentro del tráfico cifrado
WAF — Capa 7. Específico para apps web: SQLi, XSS, CSRF. Delante del servidor web
Trampa: SQLi pasa por stateful firewall (opera en L3-4, no ve payload HTTP). Necesitas WAF o NGFW
DMZ — Zona entre Internet y red interna. Con 2 firewalls: si cae el externo, la LAN sigue protegida
🚨
IDS vs IPS — Detección ▶
IDS pasivo (out-of-band) — Recibe copia del tráfico (SPAN/TAP). Solo ALERTA. No introduce latencia
IPS inline — En el camino del tráfico. Puede BLOQUEAR. Introduce latencia EXAM
NIDS — Network IDS: monitorea red entera
HIDS — Host IDS: monitorea un sistema individual
Detección por firmas — Solo detecta lo ya visto (AV clásico)
Detección por comportamiento — Detecta amenazas nuevas (zero-day). EDR/XDR
Network TAP — Copia física del tráfico. Fiable, no pierde paquetes
SPAN (Port Mirroring) — Copia por software. Puede perder paquetes bajo alta carga
📡
802.1X — NAC + RADIUS ▶
Supplicant — El cliente (PC/laptop) que pide acceso
Authenticator — El switch o AP: intermediario. NO valida credenciales EXAM
Authentication Server (RADIUS) — Quien DECIDE si se permite el acceso
Flujo: Supplicant → Authenticator → RADIUS
RADIUS — UDP 1812 (auth) / 1813 (acct). Cifra solo la contraseña
TACACS+ — TCP 49. Cifra TODO el paquete. Más seguro. Preferido para dispositivos de red
EAP — Extensible Authentication Protocol: transporta la autenticación dentro de 802.1X
🔐
VPN + TLS + Cifrado en Tránsito ▶
TLS 1.3 — Versión actual segura. Elimina algoritmos débiles (RC4, MD5)
SSL — Deprecated. No usar. HTTPS actual usa TLS, no SSL
IPSec — Capa 3. VPN site-to-site. AH (integridad) + ESP (confidencialidad)
SSL VPN — Capa 7. VPN de acceso remoto. Usa TLS. No requiere cliente especial
Split tunneling — Solo el tráfico corporativo pasa por VPN (riesgo: bypass de controles)
SFTP — SSH (puerto 22). No confundir con FTPS (TLS, puerto 990)
🏗️
SASE + Zero Trust Architecture ▶
SASE = SD-WAN + SWG + CASB + ZTNA + FWaaS. Red + seguridad convergida en la nube EXAM
SASE ≠ SIEM/SOAR/XDR — SASE es acceso y red, no detección
ZTNA — Zero Trust Network Access: reemplaza VPN tradicional
Contenedores — Aislamiento base: namespaces + cgroups. SELinux = capa adicional
Container escape — Container con root + socket Docker → escape al host
Defense in Depth — Múltiples capas de seguridad. Si falla una, las otras protegen
Secure by Design — Seguridad integrada desde el diseño, no añadida después

D4 · Security Operations
~25 preguntas | El dominio más pesado. SIEM/SOAR, IR, forense, IAM, monitoreo, pentesting
🔭
Stack de Detección: AV→EDR→XDR→SIEM→SOAR ▶
AV — Firmas conocidas. Solo detecta malware ya visto
EDR — Behavioral analysis en endpoint. Detecta zero-day. Respuesta automatizada
XDR — EDR + red + cloud + email. Visión unificada. Correlaciona todo
SIEM — Centraliza y correlaciona logs. Motor de correlación. Genera alertas EXAM
SOAR — Automatiza la respuesta. Componente central: Playbook ≠ motor de correlación (SIEM)
Flujo: AV → EDR → XDR → SIEM alerta → SOAR responde
Alert fatigue — Exceso de FP → analistas ignoran → más FN. Solución: alert tuning + SOAR
📊
Matriz TP/FP/TN/FN ▶
✅ TRUE POSITIVE
Ataque real detectado correctamente
⚠️ FALSE POSITIVE
Falsa alarma. Tráfico legítimo alertado. Causa alert fatigue
✅ TRUE NEGATIVE
Tráfico legítimo no alertado. Correcto
🚨 FALSE NEGATIVE
Ataque real NO detectado. EL MÁS PELIGROSO
FN = más peligroso (el ataque pasa desapercibido)
FP = causa alert fatigue (analistas se vuelven insensibles)
👤
IAM — Gestión de Identidad y Acceso ▶
MFA — Algo que sabes + tienes + eres. 2 de los 3 factores
TOTP — Time-based OTP (Google Auth). Válido ~30 segundos
HOTP — Event-based OTP (counter-based). No expira por tiempo
FIDO2 / WebAuthn — Sin contraseña. Llave de seguridad física (YubiKey)
SSO — Single Sign-On: un login para múltiples apps
OAuth 2.0 — AUTORIZACIÓN ("puede acceder a X en mi nombre") ≠ OIDC
OIDC — AUTENTICACIÓN sobre OAuth 2.0 ("¿quién eres?")
SAML — Federación de identidad (SSO empresarial). XML-based
PAM — Privileged Access Management: gestión de cuentas privilegiadas
RBAC — Role-Based: permisos según rol. | ABAC — Attribute-Based: según atributos
Separation of Duties — Ninguna persona tiene todo el control de un proceso crítico
Job rotation — Rota puestos. Detecta fraude por Mandatory vacation

Incident Response — NIST 800-61 ▶
1. Preparación — Políticas, playbooks, herramientas, equipo CSIRT
2. Detección & Análisis — Identificar, clasificar, priorizar el incidente
3. Contención — Aislar el sistema comprometido (corta/limitar el daño)
4. Erradicación — Eliminar malware, cuentas comprometidas, vulnerabilidades
5. Recuperación — Restaurar sistemas, verificar funcionamiento normal
6. Lecciones aprendidas — Post-incident review, mejorar procesos
Nemotecnia: PaPá Dice Cosas Limpias
Tabla inout: Contención SHORT-TERM (aislar) vs LONG-TERM (parche, cambios)
🔬
Forense Digital ▶
Order of Volatility — CPU cache → RAM → Red → Disco → Logs EXAM
Chain of Custody — Registro documental de quién toca la evidencia y cuándo
Legal hold — Preservar evidencia ante litigio. No borrar backups
Disk imaging — Copia bit a bit. Verificar con hash (MD5/SHA). Siempre en copia, nunca en original
Write blocker — Previene escrituras accidentales en evidencia original
Steganography — Oculta datos en archivos (imagen, audio). Detectado por esteganálisis
NetFlow — Metadatos (quién habla, cuánto). Sin contenido
PCAP/Wireshark — Captura completa con contenido. Evidencia forense de red
📋
Syslog Severidades + Monitoring ▶
0 Emergency — Sistema inutilizable 0 = PEOR
1 Alert — Acción inmediata requerida
2 Critical — Condición crítica
3 Error — Condición de error
4 Warning — Advertencia
5 Notice — Normal pero significativo
6 Informational — Informativo
7 Debug — Depuración. 7 = MENOS grave

UEBA — User & Entity Behavior Analytics: detecta anomalías vs baseline
TPM — Chip hardware: almacena claves criptográficas de forma segura EXAM

D5 · Security Program Management & Oversight
~18 preguntas | Gobernanza, riesgo, compliance, BCP/DRP, privacidad, frameworks
⚖️
Gestión de Riesgos — Cálculos ▶
Riesgo = Amenaza × Vulnerabilidad × Impacto
ALE = SLE × ARO (pérdida anual esperada)
SLE = Valor activo × Factor exposición (pérdida por evento)
ARO = Frecuencia anual del evento
Regla: Si control cuesta más que ALE → No implementar
Mitigar — Reducir probabilidad/impacto con controles
Transferir — Ciberseguro, outsourcing a tercero
Aceptar — Documentar y asumir. Válido si coste > impacto
Evitar — Eliminar la actividad que genera el riesgo
Riesgo inherente — Antes de controles | Residual — Después de controles
Risk appetite — Cuánto riesgo quiere asumir (estratégico, board)
Risk tolerance — Variación operativa aceptable del día a día
🏥
BIA · BCP · DRP — Continuidad ▶
BIA — Business Impact Analysis: identifica procesos críticos y su impacto si fallan
BCP — Business Continuity Plan: cómo seguir operando durante el desastre
DRP — Disaster Recovery Plan: cómo recuperar sistemas IT después del desastre
BCP = durante el desastre | DRP = después del desastre
RPO — Recovery Point Objective: máxima pérdida de datos tolerable (← pasado) EXAM
RTO — Recovery Time Objective: tiempo máximo de inactividad tolerable (→ futuro)
Nemotecnia: RPO = Punto en el pasado (datos) | RTO = Tiempo de vuelta (downtime)
Hot site — Listo para operar inmediatamente. Más caro
Warm site — Parcialmente equipado. Horas para activar
Cold site — Solo infraestructura. Días para activar. Más barato
📜
Compliance — Regulaciones ▶
GDPR — Datos personales UE. Notificación en 72h. Right to be forgotten. Multa: 4% facturación global o 20M€
HIPAA — Datos médicos EEUU. PHI protegida. BAA obligatorio. Cifrado recomendado
PCI-DSS — Datos de tarjetas de pago. Segmentación de red. Auditoría anual
SOC 2 — Auditoría de controles de seguridad en proveedores de servicios (5 Trust Criteria)
ISO 27001 — Estándar internacional para SGSI (Sistema de Gestión de Seguridad de la Información)
NIST CSF — Framework: Identify → Protect → Detect → Respond → Recover
NIST CSF orden: I-P-D-R-R (Identify, Protect, Detect, Respond, Recover)
🏛️
Gobernanza y Frameworks ▶
Gap analysis — Compara postura actual vs estándar para identificar brechas
Risk register — Lista todos los riesgos con probabilidad, impacto, owner, estado
Risk assessment — Evalúa probabilidad e impacto de cada riesgo
Cualitativo — Categorías descriptivas (Alto/Medio/Bajo). Sin cifras monetarias
Cuantitativo — Valores monetarios (SLE, ARO, ALE). Más objetivo
Auditoría interna — Hecha por personal propio. Sin valor para terceros
Auditoría externa — Hecha por tercero independiente. Requerida para compliance
Penetration test — Simulación de ataque real autorizado
🔏
Privacidad y Protección de Datos ▶
Anonimización — Irreversible. Los datos no pueden re-identificarse
Pseudonimización — Reversible con clave. Nombre → ID, guardando el mapping
Data masking — Oculta datos sensibles en entornos no productivos
Tokenización — Sustituye dato real por token. Usado en pagos (PCI-DSS)
DLP deployment — Endpoint / Network / Cloud ≠ modos de operación (inline/pasivo)
Data classification — Top Secret → Secret → Confidential → Unclassified (gobierno)
Data retention — Cuánto tiempo conservar datos según regulación
🎓
Security Awareness + Políticas ▶
Security awareness training — Formación continua al personal. El control más efectivo
Phishing simulation — Testea sin avisar. Mide click rate, mejora comportamiento
AUP — Acceptable Use Policy: qué pueden/no pueden hacer los empleados
NDA — Non-Disclosure Agreement: acuerdo de confidencialidad
MOU — Memorandum of Understanding: acuerdo no vinculante entre organizaciones
SLA — Service Level Agreement: compromisos de disponibilidad/rendimiento con proveedor
Mandatory vacation — Detecta fraude: obliga a alguien más a cubrir el puesto
⚡ Comparativas Críticas
Los conceptos que el examen usa para crear confusión
🕒 RTO vs RPO CLÁSICO
RPO — Recovery Point
Pérdida de datos máxima tolerable. Mira al pasado. Define frecuencia de backups.
RTO — Recovery Time
Tiempo de inactividad máximo tolerable. Mira al futuro. Define velocidad de recuperación.
🧂 Salt vs Pepper
Salt
Único por usuario. Público. Se almacena con el hash en la BD.
Pepper
Global (mismo para todos). Secreto. Se almacena fuera de la BD.
🚨 IDS vs IPS
IDS (pasivo)
Recibe copia del tráfico (SPAN/TAP). Solo alerta. No bloquea. No introduce latencia.
IPS (inline)
En el camino del tráfico. Puede bloquear. Introduce latencia. Si falla → fail-open o fail-closed.
🪪 OAuth vs OIDC
OAuth 2.0
AUTORIZACIÓN. "Esta app puede acceder a X en mi nombre". Tokens de acceso.
OIDC
AUTENTICACIÓN sobre OAuth. "¿Quién eres?". ID tokens. Login con Google usa OIDC.
📁 SFTP vs FTPS
SFTP
Usa SSH (puerto 22). No tiene relación con FTP clásico. Un solo canal.
FTPS
FTP + TLS/SSL (puerto 990). Es FTP clásico cifrado. Dos canales (datos + control).
💾 Rootkit vs Bootkit
Rootkit (kernel)
Vive en el kernel del OS. Reinstalar el OS = eliminado.
Bootkit
Vive en MBR/UEFI. Sobrevive a reinstalar el OS. Se activa antes de que cargue el OS.
📶 Evil Twin vs Rogue AP
Evil Twin
AP falso con el mismo SSID que una red legítima. Objetivo: MITM sobre los usuarios.
Rogue AP
Cualquier AP no autorizado en la red. No necesariamente imita a nadie.
🎟️ Kerberoasting vs Golden Ticket
Kerberoasting
Solicita TGS para SPN. Crackea offline la contraseña de cuenta de servicio.
Golden Ticket
Forja un TGT con hash de KRBTGT. Acceso total sin credenciales reales. Más poderoso.
⚙️ SIEM vs SOAR
SIEM
Detecta. Correlaciona logs. Motor de correlación central. Genera alertas.

🔐 RADIUS vs TACACS+
RADIUS
UDP 1812/1813. Cifra solo la contraseña. Para acceso de usuarios (802.1X, VPN).
TACACS+
TCP 49. Cifra todo el paquete. Más seguro. Preferido para dispositivos de red (routers, switches).
📦 Contenedores: Aislamiento
Namespaces + cgroups
Mecanismo base de aislamiento. Namespaces = paredes. cgroups = límites de recursos.
SELinux / AppArmor
Capa adicional de seguridad (MAC). Refuerza el aislamiento pero no es el mecanismo fundamental.
🏥 BCP vs DRP
BCP
Durante el desastre. Cómo seguir operando el negocio mientras el desastre ocurre.
DRP
Después del desastre. Cómo recuperar los sistemas IT a operación normal.
🔍 Vulnerability Scan vs Pentest
Vulnerability Scanning
No explota vulnerabilidades. Identifica y reporta. No destructivo. Frecuente (automatizable).
Penetration Test
Explota vulnerabilidades (con autorización). Simula ataque real. Más profundo. Puntual.
